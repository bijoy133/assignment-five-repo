Question-2: What is the difference between innerHTML, innerText, and textContent ?

Ans: The difference between innerHTML, innerText, and textContent in JavaScript. innerHTML returns or sets the HTML content inside an element, including any HTML tags and innerText It returns or sets only the visible text inside an element, ignoring hidden elements or extra spaces and textContent returns or sets all the text content of an element, including hidden elements, but without parsing HTML tags.

Question-3: What is event delegation in the DOM?

Ans: Event delegation in the DOM is a technique where a single event listener is added to a parent element, instead of adding individual listeners to each child element. This allows the parent to handle events for its child elements. The event listener on the parent element captures events that "bubble up" from child elements, allowing you to manage events for multiple elements with less code and better performance.

Question-4: What is event bubbling in the DOM?

Ans: Event bubbling in the DOM is the process by which an event, triggered on a target element, propagates upward through its ancestors in the DOM tree, from the target element to the root (document or window). This means that when an event occurs on a child element, it first runs the event handler for that element, and then the event "bubbles up" to the parent elements, allowing each parent to potentially handle the event as well.
